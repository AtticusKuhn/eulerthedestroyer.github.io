Introduction

Haskell may be my favorite programming language, but I do most of my programming on the web,where I am forced to use javascript. While I also love javascript, I do wish I could use haskell or some other functional language on the frontend. My dream is to have something like next.js, but written in a functional language. I really like next.js, especially the static building.

Features I want

  • server-side rendering
  • functional languages
  • static site generation
  • single page applications
  • minimal boilerplate

There are several things I have tried, but I haven't like any of them.

Elm

I tried elm once, but the routing didn't work also it doesn't have ssr, ssg, or spa. In fact, the first example for an elm spa has mountains of boilerplate, so I don't like it

Purescript

Purescript might be good, but there is not much tooling, and routing is bad.

Hakyll

This has static site generation and it is in haskell, but it doesn't have SSR

Conclusion

I think for the foreseeable future I will continue to use Next.js. I may prefer functional programming to javascript, but next.js is so good to use for SSR and SSG that I truly have no other options for a web framework.
